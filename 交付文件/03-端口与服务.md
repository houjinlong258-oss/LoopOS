# 端口与服务

## 结论：本项无端口，无服务端口登记

LoopOS 是 **terminal-native CLI 工具**，**不监听任何 TCP 端口**，没有前端/后端服务之分。
`AGENTS.md` 明确："No WebUI, desktop UI, automatic merge, or bypass around policy/syscall/governance gates."

因此 Crow5 全局规则中"前端端口 1001–2000 / 后端端口 2001–3000"的分配机制**不适用于本项目**，无需端口登记，也不会与其他项目端口冲突。

## 已登记端口
- 无（CLI 工具，不监听网络）

## 预览入口候选
- 无 Web 预览。终端内直接运行 `loopos --help` 查看可用子命令。
- 等价"预览"的概念在这里是 `loopos --help` 和九步 dry-run：
  ```bash
  loopos --help
  loopos run "create hello.py, run it, and confirm hello" --dry-run
  ```

## 服务与依赖说明

| 依赖类型 | 是否需要 | 说明 |
|---|---|---|
| 外部数据库 | 否 | 仅用本地 SQLite + JSONL，无 Redis/PG/MySQL |
| 外部 LLM Provider | 否（测试期） | 测试全 mock；运行时可选 OpenAI 兼容 provider，但默认本地 |
| 消息平台（Slack 等）| 否 | 仅 mock ChatOps 适配器 |
| 网络端口 | 否 | 不监听 |
| 常驻进程 | 否 | 按命令调用即起即停 |

## 与其他项目端口共存

因本项目不占用任何端口，放置在多项目工作区也不会与其他项目端口冲突。无需在 README 登记端口信息。
