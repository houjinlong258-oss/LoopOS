# 数据库交付

## 结论：无外部数据库，仅本地 SQLite + JSONL

LoopOS 在 Founding Release 阶段**不连接任何外部数据库**。
- `AGENTS.md`：所有外部 provider/gateway/database/network 在测试中全 mock
- `task_plan.md` Decisions："Database Alpha supports plans, mocks, and workspace-local samples only."

## 结构型数据库

- **SQLite（本地文件）**：用于 Memory OS（trace、skill 提案）与 Data Guard 备份库
  - adapter：`loopos/data_guard/sqlite_adapter.py`
  - 文件型 demo 命令：`loopos db sqlite-inspect / sqlite-backup / sqlite-shadow / sqlite-validate / sqlite-report`
  - demo 流程命令：`loopos db sqlite-demo --json`（在临时库上跑一遍 inspect→backup→checksum→shadow→validate→report，不碰生产）
- 无 PostgreSQL / MySQL / MariaDB

## 非结构型数据库
- 无 MongoDB 等。Agent 内部记忆用 JSONL 文件 + SQLite 索引。

## 图数据库
- 无

## 缓存数据库
- 无 Redis。本地工作区索引自带确定性检索，无需外部缓存。

## 向量数据库
- 无外部向量库。本地智能（`loopos/local_intel/`）是隐私优先的确定性索引，AST 符号 + 关键词，无 embedding 依赖。

## 本地配置文件
- `pyproject.toml`：包与依赖
- `policies/*.yaml`：Policy OS 策略包
- `examples/plugins/*/manifest.yaml`：plugin manifest
- `providers/*.yaml`：ProviderProfile

## 连接账号与凭据

- **无数据库账号密码**（无外部数据库）
- **无 LLM API key 硬编码**：测试全 mock；运行时若启用 OpenAI 兼容 provider，密钥由用户自行配置，代码不内置
- 本地 SQLite 文件无需账号，随项目目录走

## 数据安全

- Data Guard 在所有数据库 syscall 前做：危险 SQL/DSN 检测、trace 安全脱敏、本地校验备份
- 真实 migration/restore 在 Policy OS L5 下默认禁用（`task_plan.md` Decision）
- 备份库带 checksum 校验，恢复走 shadow（影子副本）+ 行数校验，不直接覆盖原库
