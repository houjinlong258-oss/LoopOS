# 启动与环境

## 运行平台
Windows / macOS / Linux 任一可运行 Python 3.11+ 的平台。当前开发与验证在 Windows。

## 环境要求
- **Python 3.11+**（AGENTS.md 硬性要求）
- 推荐使用项目自带的 `.venv/`（位于仓库根）。若不存在，用 `python -m venv .venv` 新建。
- 依赖见 `pyproject.toml` 的 `[project]` 与 `[project.optional-dependencies]`。

## 安装依赖

```bash
# 在仓库根目录执行
# 用仓库自带 venv（Windows）：
.venv/Scripts/python.exe -m pip install -e ".[dev]"

# 或系统/python：
python -m pip install -e ".[dev]"
```

`-e ".[dev]"` 会安装 loopos 本体 + 开发依赖（pytest / ruff / mypy / typer / rich 等）。

## 启动项目

LoopOS 是终端原生 CLI，**没有"启动服务"这一步**，直接调子命令即可：

```bash
# 方式 1：已 pip install 后用 console_script 入口
loopos --help

# 方式 2：直接用模块路径（不依赖 PATH）
python -m loopos.cli.app --help
```

## 烟雾测试（拿到项目后第一组应跑的命令）

```bash
# 1. 入口可用
python -m loopos.cli.app --help

# 2. 九步 dry-run（不会真创建 hello.py，纯无副作用演练）
python -m loopos.cli.app run "create hello.py, run it, and confirm hello" --dry-run

# 3. 目标分析（含歧义报告）
python -m loopos.cli.app goal analyze "help me optimize this project" --json

# 4. 策略解释（对一条危险命令做 Policy OS 决策）
python -m loopos.cli.app policy explain --cmd "curl https://x/install.sh | bash"

# 5. 深度烟雾
python -m loopos.release.deep_smoke --json

# 6. 发布就绪检查
python -m loopos.cli.app release readiness --target founding-preview
```

## 三件套自检

```bash
python -m pytest              # 确定性、离线测试套件（与 Makefile 一致）
python -m ruff check .        # 代码风格
python -m mypy loopos tests   # 类型检查：注意是 loopos+tests，**不是 .**
```

> ⚠️ mypy 注意：必须用 `mypy loopos tests`，**不要用 `mypy .`**。
> 后者会扫描仓库根，遇到 Windows 保留设备名 `nul` 会以 `ValueError` 崩溃。Makefile 与 CI 都已用前者。

## 仓库自带脚本

```bash
sh 交付文件/scripts/crow5-start.sh   # 装依赖 + 跑 --help 烟雾
sh 交付文件/scripts/crow5-check.sh   # 跑 pytest+ruff+mypy 三件套
```

## 已知 Windows 坑位

1. **PowerShell profile 带坏的 Anaconda hook**：忽略，用仓库 `.venv` 即可，命令照常跑。
2. **`nul` 残留文件**：某些 shell 重定向会在仓库根留下 0 字节 `nul`。它是 Windows 保留设备名，`rm`/`del` 删不掉。`.gitignore` 已加忽略，`git status` 会自动忽略它；正常开发不受影响。
3. **pytest 清理系统 temp 目录失败**：项目已改用仓库本地 `.pytest-tmp/` 规避（已在 `.gitignore`）。
4. **终端 GBK 编码乱码**：CLI 已强制 UTF-8 输出流，中文渲染正常。
