## 项目是什么

LoopOS 是一个**终端原生、状态机驱动、可自改进的 AI Agent 运行时**。
不是又一个 Agent，而是"运行 Agent 的内核"。自然语言只存在于输入/输出边界，
内部运行时使用 AIL 指令、AI-ISA、Policy OS 决策、syscall 路由、trace 事件、受治理内存、显式状态转移——全是结构化对象。

## 快速验活（同事拿到后执行的 3 步）

```bash
# 1. 装依赖（仓库自带 .venv 也行，或新建 venv）
python -m pip install -e ".[dev]"

# 2. 看入口是否就绪
python -m loopos.cli.app --help

# 3. 跑一个无副作用的九步 dry-run（不会真创建文件）
python -m loopos.cli.app run "create hello.py, run it, and confirm hello" --dry-run
```

## 端口说明（重点）

LoopOS 是 **CLI 工具**，**不监听任何 TCP 端口**，没有前后端服务端口之分。
`AGENTS.md` 明确："No WebUI, desktop UI, automatic merge, or bypass around policy/syscall/governance gates."
因此 Crow5 全局规则里"前端 1001–2000 / 后端 2001–3000"的端口分配机制**不适用于本项目**，本项目无需端口登记。
