# 产品设计

## 项目名称
LoopOS

## 项目一句话说明
**Not another agent. The kernel for running agents.**
终端原生、状态机驱动、可自改进的 AI Agent 运行时（runtime）。
![LoopOS - the kernel for running agents](docs/assets/brand/loopos-hero.png)

## 定位（为什么不是又一个 Agent）

大多数 AI 编码 Agent 优化的是"完成"，LoopOS 优化的是**可维护的完成**。
AI 生成的代码常常一次能跑、之后崩塌：重复逻辑、模块边界不清、隐藏全局态、弱测试、不安全工具调用、无审计、无回滚。
LoopOS 用以下治理层约束 Agent 生成的产物：
- **Policy OS**：每个动作前做结构化权限决策
- **Syscall Router**：所有外部动作是受策略门控的 syscall
- **Loop Convergence**：有界确定性调度，支持 halt/replay
- **Data Guard**：数据库操作的备份、影子、校验
- **Maintainability Gate**：代码质量治理，拒绝不可维护的补丁
- **Review Artifact**：Agent 产出的结构化评审记录
- **Trace Replay**：任意一次运行可无副作用重建

口号：传统操作系统跑程序，LoopOS 跑 Agent。

## 当前产品形态

- 纯 Python CLI 工具（Typer/Rich，附标准库 fallback）
- 终端原生，**无 Web UI、无桌面 UI、无固定端口**（AGENTS.md MVP 约束）
- 入口：`loopos.cli.app:main`（`pyproject.toml` 已注册 console_script `loopos`）
- 启动：`loopos --help` 或 `python -m loopos.cli.app --help`

## 当前已实现范围（Founding Release / Open-Source Alpha，Phase 1–15）

- 版本化 Kernel 运行、有界确定性调度、审批恢复、trace、replay
- AIL 与 AI-ISA schema，含校验与兼容适配器
- Policy OS：YAML 策略包、确定性冲突解析、审计 ID、L0–L5 安全级别
- 受策略门控的 terminal/file/git/MCP兼用/mock database syscalls
- 目标协商：低/中/高歧义三种模式
- 循环收敛：验收证据、回归、重复动作、无进展处理
- JSONL + SQLite Memory OS、受治理提案、检索、上下文预算、技能学习
- Data Guard：检测、本地校验备份库、影子计划、校验、脱敏
- 隐私优先的本地工作区索引与确定性搜索
- 隐私本地 / 混合 / 同意门控云三种算力模式
- 元数据型 plugin registry 与标准 mock Provider profiles
- 持久化 tasks/triggers/worktree leases、Producer/Verifier/Reviewer 分离
- mock ChatOps 适配器：鉴权、附件、审批、会话、投递记录
- Typer/Rich CLI + 标准库 fallback
- **Maintainability Kernel**：代码质量治理（打分/规则/门控决策）
- **System Kernel Hardening**：生命周期、不变量检查、checkpoint/replay、supervisor、signals
- **Review Artifact / Merge Gate**：结构化可评审记录 + 合并资格检查
- **Fusion Router Skeleton**：多模型 panel 选择、judge、聚合（仅 mock）
- **Prompt / Policy Distillation**：从项目规则蒸馏 behavior/renderer/policy pack
- **Boundary Adapters**：OpenAI 兼容 provider、webhook gateway、SQLite Data Guard

## 明确不做的事（边界）

- 不连接真实数据库或聊天平台
- 测试期间不做真实 provider 调用
- 不自动合并代码
- 不是操作系统级沙箱
