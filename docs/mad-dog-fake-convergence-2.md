# Mad Dog Fake-Convergence 2.0

Mad Dog attacks claims of completion across quality categories:

- fake completion;
- fake convergence;
- missing tests;
- weak design;
- brittle flow;
- implementation gap;
- token waste;
- communication noise;
- visual verification gap;
- release gap;
- security risk.

A Mad Dog finding can block delivery only when it carries evidence. This keeps
the adversary strict without turning unsupported suspicion into a release gate.
