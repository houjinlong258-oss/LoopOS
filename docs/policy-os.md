# Policy OS Kernel Contract

Policy OS evaluates typed contexts for instructions, syscalls, memory, context compilation, and rendering. A decision carries a stable ID, action, risk, approval requirement, reasons, matched rules, constraints, and audit hints.

Conflict precedence is system integrity, safety, permission/privacy, explicit user instruction, project policy, memory preference, then style and optimization. Denial overrides allow; explicit approval overrides silent execution.

Default policy packs now cover core instruction behavior, terminal safety, file/Git safety, MCP/tool routing, memory governance, skill governance, context budgets, renderer style, goal negotiation, loop convergence, and review-role separation.
