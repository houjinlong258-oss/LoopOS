# LoopOS Founding Preview Release Notes

The Founding Preview packages the first integrated, terminal-native LoopOS runtime. It combines
typed AIL, deterministic Kernel scheduling, Policy OS, governed syscalls, trace/replay, Memory OS,
Data Guard, Maintainability Gate, Review Artifact, local plugin manifests, and mock provider and
gateway boundaries.

The release is offline by default. It does not enable production database operations, real chat
platforms, automatic merge, or real model-provider traffic in tests. See
`docs/founding-preview-limitations.md` for the complete trust boundary and `CHANGELOG.md` for the
detailed change list.

## Founding Release hardening boundary

Founding Preview is the local, offline product preview. Founding Release adds trust gates around the
same runtime:

- Release readiness separates `source_tree_clean`, `packaged_artifact_clean`,
  `test_report_verified`, and `deep_smoke_verified`.
- `latest-test-report.json` is generated by `scripts/ci_report.py`; it is not a hand-written
  checklist.
- Maintainability rejects invalid or partial risky diffs instead of approving unparsed input.
- MergeGate is risk-aware: code changes without tests request changes; high-risk kernel, policy,
  data, memory, or syscall changes without tests block.
- SQLite Data Guard supports real local file inspect, backup, shadow, validate, and report flows.
- Fusion runs write trace event IDs, so multi-model mock decisions are replayable.
- Prompt Distillation v0.9 records source hashes, segment hashes, multi-label tags, activation
  workflow, and source-copy safeguards.
