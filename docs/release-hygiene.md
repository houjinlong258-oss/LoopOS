# LoopOS Release Hygiene

LoopOS is shipped as a source tree. The release artifact must be a clean, reproducible copy of the
repository without local state, credentials, caches, test scratch files, third-party source
snapshots, or planning notes.

## Blocked Content

The release checker blocks:

- VCS and local state: `.git/`, `.venv/`, `.loopos/`, `.pytest_cache/`, `.ruff_cache/`,
  `.mypy_cache/`, `__pycache__/`.
- Build outputs: `dist/`, `build/`, `*.egg-info`, compiled Python files.
- Local planning files: `task_plan.md`, `findings.md`, `progress.md`.
- Secrets and runtime data: `.env*`, `*.pem`, `*.key`, SQLite/database files, logs.
- Local reference snapshots: `OpenHands-*`, `langgraph-*`, `letta-*`, `zep-main`,
  `projectmem-*`, `hermes-agent-*`.

## Readiness Tiers

`loopos release readiness` reports four independent dimensions:

- `source_tree_clean`: the developer source tree. Advisory by default; `--strict-source` makes it
  a release blocker.
- `packaged_artifact_clean`: the staged artifact after pruning local state. Required.
- `test_report_verified`: the canonical CI report in `docs/reports/latest-test-report.json`.
  Founding Release targets require it to be generated by `scripts/ci_report.py`.
- `deep_smoke_verified`: local no-network smoke demos. Required only when `--deep` is supplied.

```bash
python -m loopos.cli.app release readiness --target founding-preview
python -m loopos.cli.app release readiness --target founding-release --strict-source --deep
python -m loopos.cli.app release readiness --target founding-release --deep --json
```

## Scripts

```bash
python scripts/check_release_clean.py --source .
python scripts/check_release_clean.py --source . --strict

python scripts/package_release.py --version 0.1.0 --output dist
python scripts/package_release.py --version 0.1.0 --output dist --no-zip
python scripts/package_release.py --version 0.1.0 --output dist --strict-source
python scripts/package_release.py --version 0.1.0 --output dist --json
```

`package_release.py` refuses to overwrite an existing staging directory. Use a fresh output
directory for repeatable release checks.

## CI Report

`latest-test-report.json` is a generated artifact, not a hand-written checklist.

```bash
python scripts/ci_report.py --run --output docs/reports/latest-test-report.json
```

When CI has already run the checks and only needs to persist a report, provide explicit counts and
statuses:

```bash
python scripts/ci_report.py --tests-passed 356 --subtests-passed 19 --tests-failed 0 --ruff passed --mypy passed
```

The report includes `generated_by`, `generated_at`, `git_commit`, Python/platform metadata, test
counts, and lint/type status. Founding Release readiness rejects stale or non-generated reports.

## Deep Smoke

```bash
python -m loopos.release.deep_smoke --json
```

The deep smoke suite runs only local deterministic flows: dry-run hello, policy explain, invalid
diff gate, SQLite file guard, and Fusion trace integration.
