## Hard-Fail Invariants (v0.2)

LoopOS v0.2 work MUST NOT violate any of the following three hard-fail
rules. Violation of any rule is detected by `scripts/anti_bloat_check.py`
and exits with non-zero status.

| # | Reason Code | Meaning | Detection |
|---|---|---|---|
| 1 | `v0_1_runtime_modified` | A file in v0.1.0 `loopos/` baseline is modified or deleted | `git ls-tree -r v0.1.0 -- loopos/` vs current working tree |
| 2 | `unauthorized_dependency` | A new dependency is added without explicit allowlist | static import scan + `pyproject.toml` / `requirements*.txt` diff |
| 3 | `v0_1_0_evidence_mutated` | A file in the v0.1.0 evidence ledger is changed | hash check against this document + tag integrity |

All other anti-bloat checks are **warning-first** (exit 0):

- `module_count_delta`
- `new_v0_2_file_over_300_loc`
- `single_use_helper_suspicion`
- `wrapper_only_function_suspicion`
- `low_deletion_ratio`
- `large_added_lines_without_paired_tests`
- `new_abstraction_count_warning`
