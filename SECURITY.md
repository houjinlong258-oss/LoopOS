# Security Policy

The Open-Source Alpha is supported on the current `0.1.x` line. It is not a production database
operator or hardened operating-system sandbox.

Report vulnerabilities privately through the repository host's security-advisory channel or another
private channel published by the maintainers. Include impact, affected version, reproduction steps,
and suggested mitigation. Do not include credentials, personal data, or active production targets.

Never use a public issue for an unpatched vulnerability. Maintainers will acknowledge a report,
triage severity, prepare tests and a fix, and coordinate disclosure. No response-time guarantee is
made during Alpha.

Real database operations, automatic merge, unrestricted plugins, and real ChatOps/provider calls are
disabled in Alpha. Policy denials must not be bypassed to demonstrate a vulnerability.
